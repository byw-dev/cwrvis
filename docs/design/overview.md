# 系统总体设计 — cwrvis

> 文档版本：v1.0  
> 阶段：第一阶段 MVP

---

## 系统定位

cwrvis 是一套气候/水资源格点数据可视化与区域统计分析系统，采用 B/S 架构。第一阶段目标为快速交付可演示版本，满足甲方验收要求。

系统的核心设计原则是**预计算优先**：将所有计算密集型任务（netcdf 解析、区域统计、报告生成）移到离线脚本阶段完成，运行时只做数据读取和展示，以最大限度降低后端复杂度和服务器资源要求。

## 数据流

### 格点可视化数据流

```
用户选择 var + 时间 + 颗粒度
    │
    ▼
前端拼接 URL：/grid/{gran}/{var}.json
    │
    ▼
fetch JSON（浏览器缓存友好）
    │
    ▼
传入 Web Worker
    │
    ▼
双线性插值 → 色卡映射 → 阈值过滤 → ImageBitmap
    │
    ▼
MapLibre ImageSource 更新 → 渲染到地图
```

### 区域统计数据流

```
进入区域统计模块，或切换区域/var
    │
    ├─ GET /api/v1/stats?region_id=X&granularity=year&year_start=2000&year_end=2025&var=CWR
    └─ GET /api/v1/stats?region_id=X&granularity=month&year_start=2000&year_end=2025&var=CWR
    │  （两个请求并行）
    ▼
前端缓存 year + month 原始数据到 Pinia store
    │
    ├─ 聚合模式=逐年/逐月：直接取缓存数据渲染地图色块 + Inspector
    └─ 聚合模式=年平均/月平均/季平均：客户端从缓存计算均值
    │
    ▼
[查看历史] → ECharts 折线图（支持追加多变量叠加对比）
```

### 报告下载数据流

```
用户选择区域 + 颗粒度 + 时间范围 → 点击下载
    │
    ▼
GET /api/v1/report/download?region_id=X&granularity=year&start=2000&end=2024
    │
    ▼
FastAPI 按命名规则拼接文件路径 → 检查文件存在 → 返回文件流
（或 302 重定向到 Nginx 静态路径）
```

## 关键约束与决策记录

### ADR-001：格点数据以 JSON 静态文件分发，不提供实时 API

**决策**：格点数据预生成为独立 JSON 文件，由 FastAPI StaticFiles 直接托管，前端直接 fetch。

**原因**：
- 消除运行时 netcdf 解析，零计算压力
- 浏览器 HTTP 缓存天然生效，重复访问同一帧无网络开销
- 部署与运维极简，无需专门的数据服务

**代价**：
- 预生成脚本运行一次需要较长时间（可接受，离线完成）
- 磁盘占用约 800MB（可接受）
- 数据更新需要重新生成（第一阶段数据不更新，可接受）

### ADR-002：区域统计使用 SQLite，不使用 PostgreSQL

**决策**：使用 SQLite 存储区域统计数据。

**原因**：
- 数据量约 16 万行，SQLite 完全胜任
- 无需独立数据库服务，部署简单
- 只读访问为主，无并发写入问题

### ADR-003：报告文件预生成，不动态生成

**决策**：所有报告 `.docx` 文件由同事离线生成，系统只负责按命名规则提供下载。

**原因**：
- 完全消除后端 Word 文档生成的复杂性
- 动态生成报告需要图表渲染，实现成本高
- 固定组合的报告数量有限，预生成可行

### ADR-004：底图可切换，默认 OpenStreetMap

**决策**：底图 provider 通过配置切换，默认使用 OpenStreetMap（无需 API Key），可选高德街道图、高德卫星图、Carto Dark。

**原因**：
- OSM 开箱即用，无 Key 申请流程，降低部署门槛
- 部分场景（对外演示、无高德配额时）需要备用底图
- 切换成本极低：`config/basemaps.ts` 统一维护，底图 URL 和坐标系标记集中定义

**坐标系处理**：格点数据为 WGS-84，区域 GeoJSON（来自高德）为 GCJ-02。
- WGS-84 底图（OSM/Carto）：格点直接使用原始坐标；GeoJSON 整体偏移 −0.01° lon / +0.005° lat 修正到 WGS-84
- GCJ-02 底图（高德）：格点整体偏移 +0.01° lon / −0.005° lat；GeoJSON 直接使用
- 1° 格点精度约 100km，GCJ-02 偏移约 500m，视觉不可见，整体修正可接受
